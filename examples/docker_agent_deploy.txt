Docker容器本地部署AI Agent完全指南（2025）

Docker在2025年推出了完整的AI Agent本地部署工具链，主要包括五大核心工具：Docker Model Runner（本地运行LLM模型，兼容OpenAI API）、Docker MCP Gateway（通过Model Context Protocol连接外部工具和服务）、Docker cagent（声明式YAML多Agent运行时，无需手写代码）、Docker Offload（GPU加速云端环境）和Docker Compose（定义和运行多容器AI应用的统一编排工具）。

方案一：使用Docker Model Runner加自定义Agent。适合需要自己编写Agent逻辑的Python开发者。首先启用Docker Model Runner，通过docker desktop enable model-runner --tcp 12434命令开启服务。然后拉取本地模型如docker model pull ai/smollm2。可选的模型包括ai/llama3.2、ai/deepseek-r1-distill-llama、ai/gemma3等。Python代码通过HTTP请求调用本地的localhost:12434端口，发送OpenAI兼容的API请求。完整示例项目有dockeraiagent，基于Streamlit的AI Agent可一键docker compose up启动。另一个示例MicrosoftAgentFrameworkWithDockerModelRunner使用Microsoft Agent Framework加DMR的本地聊天Agent。

方案二：使用Docker cagent，这是推荐的零代码方案，适合不想手写代码的用户。通过YAML声明式定义Agent，在agents下可以定义root主Agent、researcher研究子Agent、writer写作子Agent等多个角色。每个Agent可配置model模型、instruction指令和toolsets工具集。工具集通过MCP协议引用，如docker:duckduckgo搜索工具。运行命令为cagent run agent.yaml，还可以通过cagent push推送到Docker Hub分享，其他机器cagent pull拉取运行。

方案三：Docker Compose多服务Agent架构。适合需要集成模型、工具网关、Web UI的完整Agent应用。典型的compose.yaml包含三个部分：adk服务（Agent应用的Web界面，监听8080端口）、mcp-gateway服务（MCP工具网关，通过SSE传输协议连接duckduckgo等工具服务器）和models声明（如gemma3模型，配置model名称和context_size上下文大小）。一键启动只需docker compose up --build。Docker官方示例compose-for-agents包含Auditor审查者、Critic批评者、Reviser修订者等多Agent协作。

方案四：Goose加Docker的简易Agent方案，适合快速构建私有的带浏览器界面的AI助手。技术栈包括Goose开源AI助手框架、Docker Model Runner运行本地模型、Docker MCP Gateway连接外部工具如YouTube字幕提取，以及ttyd通过浏览器访问终端Agent。访问http://localhost:7681即可使用。

硬件要求方面，ai/smollm2仅需约1GB显存和0.5GB存储，适合测试。ai/gemma3:4B-Q4_0需要3.5GB显存和2.31GB存储，是推荐配置。ai/qwen3-30b需要16GB以上显存和约20GB存储，需高端GPU。ai/llama3.2约需2GB显存和2GB存储，是平衡选择。

关键技术要点包括：Docker Model Runner默认监听localhost:12434，容器内使用model-runner.docker.internal。MCP协议通过--servers参数启用不同工具如duckduckgo、context7、filesystem。安全沙盒方面，Agent生成的代码应在隔离容器中运行，可禁用网络防止数据泄露。混合AI模式下，本地小模型处理常规任务，远程大模型处理复杂推理，可降低5至30倍成本。

推荐学习路径为：先安装Docker Desktop 4.40以上版本并启用Model Runner，拉一个轻量模型如ai/smollm2测试，用cagent run体验零代码Agent，克隆compose-for-agents学习完整多Agent架构，最后自定义YAML配置添加自己的MCP工具和服务。
